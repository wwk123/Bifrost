# 任务概述
- **任务名称**: 社交化收益竞赛平台实现
- **相关文档**: `design-system.md`, `社交化收益竞赛_最终实施方案.md`
- **目标**: 按照设计规范与实施方案，交付可运行的前端应用，提供收益竞赛的可视化、社交化体验。

# 背景要点
- 黑客松赛道二项目，主攻 Bifrost 液态质押用户增长。
- 技术栈推荐: Next.js 14、TypeScript、TailwindCSS、Framer Motion、Recharts、Zustand、TanStack Query。
- 核心模块: 收益排行榜、成就系统、策略分享、段位体系、挑战任务、推荐奖励、分享卡片等。

# 约束与要求
- 保持代码模块化，避免单文件超过 1000 行。
- 新增功能必须配套测试与基础校验。
- 严格遵守设计系统中的色彩与排版规范。
- 文档、注释与用户沟通统一使用中文。

# 风险与应对
- **前端框架搭建耗时**: 使用官方脚手架快速初始化，优先保证目录清晰。
- **数据来源暂缺**: 阶段性采用 Mock 数据，抽象接口方便后续接入真实 API。
- **时间紧张**: 分阶段交付，先确保核心用户路径可演示，再迭代增强功能。

# 阶段拆解
1. **基础设施阶段**
   - 创建项目脚手架与统一配置（ESLint、Prettier、Tailwind、Commit hooks）。
   - 落地设计系统中的色彩、字体、阴影等 Token。
2. **核心功能阶段**
   - 实现收益排行榜、用户卡片、成就系统 UI 与交互。
   - 策略分享、段位展示、挑战任务等主要组件。
   - Mock 数据与状态管理（Zustand + TanStack Query）。
3. **增强体验阶段**
   - 动画、过渡、分享卡片生成。
   - 响应式布局、移动端适配。
   - 数据统计图表、推荐奖励流程。
4. **测试与交付阶段**
   - 单元测试、组件测试及端到端关键路径验证。
   - 文档更新、运行说明、部署准备。
